Pokémon Project Recreation

An older Unity project focused on object-oriented design

Introduction

This project was an attempt to recreate the mechanics of Pokémon in Unity. It is an older project and unfortunately the source code is no longer available. Despite this, the design focused heavily on object-oriented programming principles, creating a flexible and modular system.

Project Design

Each Pokémon, along with its characteristics, abilities, and attacks, was managed through dedicated classes. Scriptable Objects were used extensively to store reusable data such as stats, moves, and evolution lines.

LINQ queries allowed for easy filtering and selection of Pokémon based on type, level, or abilities, making the battle and management systems efficient and readable.

The focus on object-oriented design made it easy to expand the project with new Pokémon, moves, or mechanics without rewriting existing systems.

Technical Notes

Although the project is old and the code is lost, the approach used can be summarized as follows:

  • Object-Oriented Design: Each Pokémon, move, and trainer was represented as an object with encapsulated behavior.
  • Scriptable Objects: Used to store data-driven configurations for Pokémon stats, evolutions, and attacks.
  • LINQ: Applied for filtering and querying collections, like selecting Pokémon by type or ability.
  • Battle System: Designed to handle multiple Pokémon and attacks.

Conclusion

This project helped me learn advanced object-oriented programming concepts in Unity, including data-driven design with Scriptable Objects and functional querying with LINQ. Although the project is no longer maintained, it served as an important step in understanding modular and scalable game architecture.
